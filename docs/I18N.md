# Internationalization (i18n)

The app uses **Transloco** for translations and supports **English (en)** and **Arabic (ar)**. Arabic is treated as RTL.

## Configuration

Configured in `src/app/app.config.ts`:

- **provideTransloco** – Provides Transloco with:
  - **config.availableLangs** – From `core/constants/app` (`AvailableLangs`: `['en', 'ar']`).
  - **config.prodMode** – `!isDevMode()`.
  - **loader** – `TranslocoHttpLoader` (from `core/injections/transloco-loader.ts`), which loads JSON via HTTP.

Loader behavior:

- **TranslocoHttpLoader.getTranslation(lang)** – Fetches `/i18n/${lang}.json`. The app serves files from `public/i18n/`, so the requested URL is typically `/i18n/en.json` or `/i18n/ar.json`.

## Translation Files

| Language | File | Direction |
|----------|------|-----------|
| English | `public/i18n/en.json` | LTR |
| Arabic  | `public/i18n/ar.json` | RTL |

Files are part of the `public` assets and are copied as-is by the build. Add or update keys in both files to keep translations in sync.

## Usage

- **In templates:** Use the `transloco` pipe or `transloco` directive, e.g. `{{ 'common.home' | transloco }}` or `transloco="common.home"`.
- **In TypeScript:** Inject `TranslocoService` and call `this.translocoService.translate('key')` or `this.translocoService.selectTranslate('key')`.

Never hardcode user-facing strings in templates or TS; use translation keys.

## Key Naming

- Use **semantic, namespaced** keys, e.g.:
  - `common.home`, `common.search`, `common.submit`
  - `auth.login.button`, `home.hero.title`
- Keeps keys organized and avoids collisions.

## Language and Locale

- **provideAppLang** (`core/injections/app-lang.ts`) – Provides:
  - **APP_LANG_TOKEN** – Resolved from cookie (`AppConstants.LANG_KEY`); falls back to `AppConstants.DEFAULT_LANG` if missing or invalid.
  - **LOCALE_ID** – Set to `'ar-EG'` for Arabic (after registering `localeArEG`) and `'en-US'` for English (after registering `localeEnUS`). Used by Angular pipes (e.g. date, number).

Language constants:

- **Language** – `core/constants/app/language.constants.ts`: `ENGLISH: 'en'`, `ARABIC: 'ar'`.
- **AvailableLangs** – `['en', 'ar']`.

Switching language should update the cookie, Transloco active lang, and document direction (RTL for `ar`). Layout and components must work in both directions; prefer logical CSS and RTL-aware styling (see [STYLING_AND_THEMING.md](STYLING_AND_THEMING.md)).
